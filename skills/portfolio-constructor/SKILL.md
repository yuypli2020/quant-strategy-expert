# portfolio-constructor — 多股组合构建 V27

> **版本**：V27  
> **定位**：接收多个标的的裁决分 → 输出最优资本配置方案  
> **触发**：分析超过1只股票时自动调用 | 用户说「怎么分配仓位」「构建组合」  
> **前置**：genesis-v22 分析结果（各标的综合分 + 建议仓位）

---

## 核心设计理念

```
传统做法：每只股票独立分析，仓位平均分
V27 做法：基于置信度差异分配，资金向高置信度标的集中

核心原则：
1. 置信度高的标的 → 更大仓位
2. 高相关板块 → 分散配置，不押注单一主题
3. 区间相位决定总仓位上限
4. ATR止损决定单股最大亏损
```

## 第三步：置信度评级

```
置信度分档（基于综合分）：

| 分档 | 综合分 | 含义 | 仓位权重系数 |
|------|--------|------|------------|
| A级 | 4.5-5.0 | 高置信 | ×1.0 |
| B级 | 4.0-4.4 | 中高置信 | ×0.8 |
| C级 | 3.0-3.9 | 中置信 | ×0.6 |
| D级 | 2.0-2.9 | 低置信 | ×0.3 |
| F级 | <2.0 | 不推荐 | ×0（不纳入）|

* 量化严谨派判定为"不可信"的标的，降一档处理
```

---

## 第四步：板块集中度检查

```
同一板块仓位上限：
  - 单一板块最大权重 ≤ 40%（避免过度集中）
  - 若某板块合计权重 > 40% → 超配部分强制分散到其他板块

高相关板块（合并计算）：
  [电子, 通信, 计算机] → 归为"科技"大类，上限40%
  [煤炭, 有色, 钢铁] → 归为"周期"大类，上限30%
  [银行, 非银] → 归为"金融"大类，上限30%
```

---

## 第五步：仓位分配计算

### 公式

```
单股初始权重 = (标的建议仓位 × 置信度系数) / Σ(所有标的建议仓位 × 各自置信度系数)

调整后权重 = min(初始权重, 板块上限) × 区间仓位系数

最终仓位 = 调整后权重 / Σ(所有调整后权重) × 可用总资金
```

### 示例计算

```
标的A: 建议50% × 置信度A级(1.0) = 50
标的B: 建议30% × 置信度B级(0.8) = 24
标的C: 建议20% × 置信度C级(0.6) = 12
Σ = 86

初始权重: A=50/86=58%, B=28%, C=14%

若A属于"电子"板块，当前有B也属于"电子"
电子合计 = 58%+28% = 86% > 40%上限
强制调整: 电子最多40%，超配46%分散到C
→ 最终: A=30%, B=10%, C=44%

区间仓位: Phase 3，上限70%
→ 最终可用仓位 = 70% × [A=30%, B=10%, C=44%]
```

## 第六步：风险验证

### 单股风险验证

```
单股最大亏损 = |入场价 - 止损价| × 仓位金额

风险堡垒检查：
  - 单股最大亏损 > 账户总资产 × 3% → 降仓
  - 单股最大亏损 > 账户总资产 × 5% → 不建议建仓

组合风险验证：
  - 组合VaR(95%, 1日) < 总资产 × 2% → 通过
  - 组合最大回撤预估 < 总资产 × 15% → 通过
```

### 关联风险检查

```
同板块标的数量 > 3 → 警告（过度集中）
止损位相关性：两只标的止损同时触发的概率
  - 若是 > 30% → 降低其中一只仓位
```

## 第七步：再平衡触发

```
触发条件（任一满足即触发再平衡）：

| 触发类型 | 条件 | 操作 |
|---------|------|------|
| 偏离触发 | 单股偏离目标仓位 > 5% | 再平衡 |
| 止损触发 | 触及止损位 | 退出 + 资金再分配 |
| 止盈触发 | 达到目标位 | 减仓或持有 |
| 季度触发 | 每季度末 | 全组合审视 |
| 区间变化 | 区间相位改变 | 重新评估总仓位上限 |

再平衡优先级：
1. 止损触发 → 立即退出
2. 偏离触发 → 按月度频率处理
3. 季度审视 → 按季度频率处理
```
