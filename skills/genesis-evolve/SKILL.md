# Genesis-Evolve — P2 进化沉淀

## 核心流程

```
用户说「复盘」「证伪」「XX为什么跌了」
  → Step 1: 调取历史报告（本地 + IMA）
  → Step 2: 拉取实际走势数据
  → Step 3: 归因分析（四个维度）
  → Step 4: 提炼规则（新规则 / 修正已有规则）
  → Step 5: 等待人工确认
  → Step 6: 写入 IMA 知识库（如果可用）或本地 strategies/
```

## Step 1：调取历史报告

### 本地报告
```bash
ls ~/.openclaw/workspace/strategies/<strategy_name>/v<N>/
```

### IMA 运行日志（如果可用）
搜索 IMA Genesis 知识库的 `02_运行日志/evolution_log/`。

## Step 2：拉取实际走势数据

```bash
westock-data kline <代码> --period day --limit <事后N天>
```

## Step 3：归因分析

必须覆盖四个维度：

| 维度 | 检查要点 |
|------|----------|
| **技术面** | MA/RSI/MACD 是否失效？ |
| **基本面** | 是否有未预见的财报/公告/政策？ |
| **资金面** | 主力资金是否逆向操作？ |
| **情绪面** | 市场整体情绪是否逆转？ |

## Step 4：提炼规则

### 新规则（之前没有对应规则）
在 `strategies/<strategy_name>/v<N+1>/` 下创建新文件，或从 IMA 知识库创建。

**文件命名**：`evo_XXX_<简短描述>.md`

### 修正规则（已有规则但需调整）
1. 读取对应规则文件
2. 在文件末尾添加 `## 修订记录`
3. 记录修订内容、日期、原因

## Step 5：等待人工确认

**⚠️ 规则写入前必须人工确认**

输出提炼出的规则内容，并提示：
> 「根据复盘归因，建议新增/修正以下规则：
> 1. [规则名] — [内容摘要]
> 
> 请确认是否写入？（回复「确认」或提出修改意见）」

## Step 6：写入

### 如果 IMA 可用
使用 `ima` skill 上传规则文件到 IMA Genesis 知识库。

### 如果 IMA 不可用
保存到 `strategies/<strategy_name>/v<N+1>/`，并更新 `strategies/index.json`。

## 关键原则

1. **诚实地证伪**：不要为错误的分析找借口
2. **规则具体可执行**：必须包含明确的触发条件和判断逻辑
3. **小步迭代**：每次只新增/修正 1-2 条规则
4. **人工把关**：规则写入必须经过人工确认
