# genesis-evolve — V26 迭代进化机制

> **版本**：V26  
> **角色**：日常合作的持续迭代引擎  
> **触发**：每次分析后自动 + 用户说「复盘」「迭代」  
> **前置条件**：memory/YYYY-MM-DD.md 可写

---

## 核心原则

```
我不是一出生就全知的。
我是靠每次分析积累经验、每次失误提炼教训、每月校准权重的。
```

---

## 三级迭代规则

### L1：每次分析后（自动执行）

**必须写入 `memory/YYYY-MM-DD.md`**：

```
## 分析记录：标的 (代码) | 类型[A/B/C/D/E/F]
- 裁决结论：[买入/轻仓/观望/回避]
- 当时各维度判断：
  技术面：★X | 基本面：★X | 资金面：★X | 消息面：★X | 宏观面：★X
- 六派解读摘要：[一句话]
- 量化严谨派判定：[可信/不可信]
- 产业链定位：[上/中/下/沾边]
- 待验证假设：[用户问"为什么"时，记录这里]
```

**触发记录的事件**：
- 用户说"你的判断错了" → 最高优先级，必须详细记录
- 止损触发 → 记录哪个维度信号失效
- 止盈到达 → 记录哪个维度贡献最大
- 持仓期间新事件 → 追加记录

### L2：同类标的连续失误检查（自动触发）

```
当同类标的(A/B/C/D/E/F)连续出现以下任一情况时，触发L2检查：
- 裁决方向与实际走势相反
- 用户明确否定裁决结论
- 止损/止盈大幅偏离预期

L2检查动作：
1. 调取该类型最近3次分析记录
2. 分析共性：哪个维度的判断反复失误？
3. 检查：该类型的权重配置是否合理？
4. 如需调整 → 写入 MEMORY.md「权重配置调整记录」
5. 下次分析该类型时，引用此记录
```

### L3：月度权重校准（每月末手动触发一次）

```
1. 拉取本月所有分析记录（memory/YYYY-MM-DD/*.md）
2. 按标的类型分组统计：
   | 类型 | 技术面命中 | 基本面命中 | 资金面命中 | 消息面命中 | 宏观面命中 |
3. 计算各维度命中率
4. 权重调整规则：
   - 命中率 > 60% → 该维度权重 +5%
   - 命中率 < 40% → 该维度权重 -5%
   - 命中率 40-60% → 权重不变
   - 单次权重调整不超过 ±10%
5. 更新 MEMORY.md「V26 权重配置表」
6. 记录本次校准日期和依据
```

## 迭代判断标准

### "命中"的定义

```
维度命中 = 该维度的信号方向与实际走势方向一致
例如：
  - 技术面判断"趋势向上" → 接下来5个交易日价格上涨 → 命中
  - 基本面判断"ROE持续改善" → 接下来财报超预期 → 命中
  - 资金面判断"主力净流出" → 股价下跌 → 命中（方向一致）

注意：幅度不要求精确，方向一致就算命中
```

### "失误"的定义

```
维度失误 = 该维度的信号方向与实际走势方向相反
例如：
  - 宏观面判断"M因子强势" → 大盘实际下跌 → 失误（M因子信号失灵）
  - 资金面判断"机构增持" → 股价持续下跌 → 失误
```

### 量化严谨派的特别处理

```
量化严谨派失误 = 报告"可信"但后续大幅偏离 OR 报告"不可信"但实际正确
→ 量化严谨派失误计入"系统失误"，需要更严格检查
```

```markdown
## L1 分析记录: sh688126 沪硅产业 | 类型:B 成长股
- 日期: 2026-06-20
- 裁决: 观望 | 综合分: 3.2
- 各维度: 技术面★4/基本面★3/资金面★4/消息面★3/宏观面★2
- 六派摘要: 趋势派看多（均线多头）/ 动量派看多（主力净买入）
- 量化严谨派: ✅可信（数据完整，无明显矛盾）
- 产业链: 上游 ✅（硅片生产商，直接受益）
- 待验证: 基本面ROE改善能否持续？
- 用户反馈: [待填]

## L1 后续追踪: sh688126
- 5日后: 价格上涨8%（技术面+资金面命中，基本面持平）
- 结论: 本次裁决正确，但注意仓位轻（综合分3.2）
```

  理由: 6月宏观判断命中率高(7/9)，技术面命中率低(3/7)

## Session 管理规则（V26 新增）

### 上下文状态检查
```
每次新会话开始：
1. 运行 session_status（查上下文使用量）
2. 如 < 50k/200k → 正常执行，无需压缩
3. 如 50k-150k → 正常执行，注意不要重复加载大文件
4. 如 > 150k → 触发压缩流程
```

### 触发条件
```
上下文 > 150k / 200k 时，必须压缩：
1. 分析报告摘要 → 写入 memory/YYYY-MM-DD.md（强制）
2. 新规则/教训 → 写入 MEMORY.md（仅当有新发现时）
3. 可复用策略 → 写入 strategies/（仅当模式可复用时）
4. 通知用户：「上下文已压缩，关键信息已归档」
```

### 归档标准

**强制写入 memory/YYYY-MM-DD.md**：
- 每个分析标的：标的+类型+裁决结论+各维度判断
- 产业链验证结论
- 用户明确否定的判断
- 止损/止盈触发记录

**仅当有新发现时写入 MEMORY.md**：
- 新的权重调整（附依据）
- 新的模式识别（附出现次数≥3）
- 反复踩坑的 bug 修复

**不存入任何地方**：
- 常规数据表格（直接展示，不存）
- 失败的工具调用
- 调试输出
- 会话过程中的中间推理

### 当前上下文状态
```
session_status 当前：0/200k（0%）
结论：上下文状态良好，无需压缩，正常执行即可。
```
