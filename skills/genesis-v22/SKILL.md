# genesis-v22 — Genesis V26 分析裁决系统

> **版本**：V26  
> **定位**：灵活多维分析框架 + 按标的类型自适应权重  
> **触发**：用户说「分析 XX」「裁决」「综合评估」  
> **前置条件**：`westock-data` 必须可用  
> **V26 核心升级**：五维观察面 + 六派分析 + 标的类型自适应 + 迭代进化机制

---

## 核心哲学转变

```
V22.1: 缠论独裁（40%权重，无视其他）
V24.1: 六框架均等弱加权（无灵活区分）
V25.0: 五派风险辩论（风险层独立，但仍是静态权重）
V26:    标的类型自适应 + 五维灵活组合 + 迭代进化
```

**V26 的核心洞察**：

1. **不同股票类型，分析重点不同**
   - 蓝筹股/大盘股 → 基本面60% + 宏观面30% + 技术面10%
   - 成长股/题材股 → 技术面50% + 资金面30% + 基本面20%
   - 周期股 → 宏观面50% + 基本面30% + 技术面20%
   - 价值股 → 基本面50% + 资金面25% + 技术面25%

2. **五个观察面必须全部扫描，但权重按标的类型分配**
   - 技术面 / 基本面 / 资金面 / 消息面 / 宏观面

3. **六派分析是解读工具，不是独立评分**
   - 六派不直接投票，而是解释"为什么这个维度的信号强/弱"
   - 量化严谨派有否决权：说"信号不可信" → 必须采纳

## 第一步：标的类型识别（必须首先执行）

```bash
# 识别标的类型
westock-data profile <代码>
westock-data quote <代码>   # 看市值、PE
westock-data finance <代码> --num 4  # 看ROE趋势、营收增速
```

### 标的类型分类规则

| 类型 | 识别条件 | 核心问题 |
|------|---------|---------|
| **A. 蓝筹大盘股** | 总市值 > 500亿 且 PE>0 且 ROE>10% | 估值合理吗？宏观受益吗？ |
| **B. 成长题材股** | 总市值 50-500亿 且 营收增速>20% | 趋势强吗？资金跟进吗？ |
| **C. 周期股** | 行业属于 煤炭/有色/钢铁/化工/猪肉/航运 等强周期 | 宏观/商品周期在哪？ |
| **D. 价值低估股** | PB<1.5 且 PE>0 且 ROE>8% 且 市值>50亿 | 真的是低估还是价值陷阱？ |
| **E. 困境反转股** | 连续2年净利润下滑 或 ST/Risk | 困境来源清楚吗？拐点信号？ |
| **F. 未知/混合型** | 不满足以上任一条件 | 先扫描全部维度 |

### 各类型权重配置表

```
         技术面  基本面  资金面  消息面  宏观面
A.蓝筹    10%    40%    15%    10%    25%
B.成长    40%    15%    30%    10%     5%
C.周期    15%    25%    10%    15%    35%
D.价值    20%    40%    25%    10%     5%
E.困境    20%    30%    20%    25%     5%
F.未知    25%    25%    20%    15%    15%
```

## 第二步：五维观察面数据采集

### 2a：技术面

```bash
westock-data kline <代码> --period day --limit 260
westock-data kline <代码> --period week --limit 80
westock-data technical <代码> --group ma,macd,rsi,boll
```

技术面评估要点：
- 趋势方向（MA排列）
- 动量（MACD开口方向）
- 超买超卖（RSI位置）
- 相对位置（BOLL通道）
- 成交量配合

### 2b：基本面

```bash
westock-data finance <代码> --num 12
westock-data quote <代码>   # PE/PB/市值
westock-data dividend <代码> --years 3  # 分红稳定性
```

基本面评估要点（CAN SLIM 七因子）：
- C（Current EPS）：当季EPS同比增速
- A（Annual EPS）：年度EPS增长趋势
- N（New highs）：是否接近52周高点
- S（Supply/Demand）：成交量放大配合
- L（Leader/Laggard）：相对板块强弱
- I（Institutional）：机构持股变化
- 额外：ROE质量、现金流、商誉

### 2c：资金面

```bash
westock-data asfund <代码>
westock-data shareholder <代码>
westock-data chip <代码>   # 仅沪深A股
```

资金面评估要点：
- 主力净流入/流出（5日/20日趋势）
- 机构股东增减
- 北向资金（沪股通/深股通持股变化）
- 筹码集中度（成本分布）

### 2d：消息面

```bash
westock-data news <代码> --limit 15 --type 2
westock-data notice <代码> --limit 5
westock-data marketnews hs   # 板块整体新闻
```

消息面评估要点：
- 催化剂类型（政策/业绩/产品/事件）
- 消息强度（顶层政策 > 行业 > 公司）
- 预期差（是否超预期）
- 时机（是否已定价）

### 2e：宏观面

```bash
westock-data sector --rank interval_chg_rank_sw1 --sort chg5Days
westock-data kline sh000300 --period day --limit 60   # M因子
westock-data calendar --limit 10 --country 1 --indicator 2
```

宏观面评估要点：
- 大盘方向（M因子，仓位上限决定）
- 板块相对强弱
- 政策/产业周期位置
- 宏观数据（PMI/商品价格等对周期股关键）

## 第三步：六派解读（解释"为什么"，不直接投票）

| 流派 | 主要看 | 核心问题 | 对哪类股票最准 |
|------|--------|---------|---------------|
| **趋势派** | 技术面+宏观面 | "趋势形成了吗？还在吗？" | 成长股、趋势蓝筹 |
| **回归派** | 技术面+基本面 | "偏离均值太多了吗？" | 价值股、蓝筹 |
| **动量派** | 技术面+资金面 | "资金在持续买入吗？" | 成长股、题材股 |
| **情绪派** | 消息面+资金面 | "市场情绪过头了吗？" | 所有类型，辅助 |
| **周期派** | 宏观面+基本面 | "宏观/商品周期在哪阶段？" | 周期股、困境反转 |
| **量化严谨派** | 全部维度 | "信号统计可信吗？" | **所有类型，有否决权** |

**量化严谨派判定标准**：
```
可信：样本量足够(n>30) + 统计显著(p<0.05) + 非幸存者偏差 + 数据质量高
不可信：样本量不足 / 指标间矛盾 / 明显过拟合 / 数据缺失
→ 如不可信：必须降低该维度权重，并在报告中明确说明
```

## 第四步：M因子快检（大盘仓位上限）

```bash
westock-data kline sh000300 --period day --limit 60
```

| M评分 | 大盘状态 | 仓位上限 |
|-------|----------|----------|
| 8-10 | 强势 | 100% |
| 6-7 | 偏强 | 80% |
| 4-5 | 中性 | 60% |
| 2-3 | 偏弱 | **30%** ⚠️ |
| 0-1 | 弱势 | **10%** 🔴 |

M < 4 → 仓位上限30%，是宪法级硬约束。

---

## 第五步：融合裁决 + 权重组合

```
最终综合分 = Σ(各观察面得分 × 该类型权重)

裁决判定：
  综合分 ≥ 4.0 + 量化严谨派可信 → 买入（按仓位上限）
  综合分 3.0-3.9 + 量化严谨派可信 → 轻仓
  综合分 < 3.0 或 量化严谨派不可信 → 观察/回避
```

## 迭代进化机制（V26 新增核心模块）

### 日常迭代规则

**规则1：每次分析后记录**
```
每次交付裁决报告后，必须写入 memory/YYYY-MM-DD.md：
- 分析标的 + 裁决结论
- 哪些维度的判断被后续走势验证/打脸
- 新发现的模式（出现3次→提升到规律）
```

**规则2：同类型标的连续失误触发检查**
```
同类标的（A/B/C/D/E/F）连续失误 ≥ 2次：
→ 检查：该类型的权重配置是否合理
→ 可能需要调整（证据 → MEMORY.md → 未来生效）
```

**规则3：用户反馈是最高优先级**
```
用户说「你判断错了」「XX因素你没考虑到」
→ 必须记录到 MEMORY.md「反复踩过的坑」
→ 下次分析该类型时必须引用
```

**规则4：月度权重校准**
```
每月末：
1. 拉取本月的所有裁决记录
2. 按标的类型统计：各维度命中/失误率
3. 命中率 >60% 的维度 → 权重+5%
4. 命中率 <40% 的维度 → 权重-5%
5. 更新 MEMORY.md 中的权重参考表
```

**规则5：季节性模式识别**
```
周期股（A.2）：宏观面权重在宏观上行期自动+10%
成长股（B）：动量派信号在牛市加速期权重+10%
价值股（D）：回归派信号在熊市防御期权重+10%
```
