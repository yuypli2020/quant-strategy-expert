# genesis-v22 — Genesis 融合派策略系统 V22.1

> **版本**：V22.1  
> **定位**：四框架共振 + 缠论优先体系  
> **触发**：用户说「分析 XX」「融合分析」「四框架共振」  
> **前置条件**：`westock-data` 必须可用

---

## 系统概述

Genesis V22.1 是四框架共振分析系统。分析标的时，同时运行四个独立框架评估，最终融合评分并输出操作建议。

**与 V24.x 的区别**：V22.1 是单标的四框架分析，V24.x 是多因子裁决系统。两者独立运行，用户说「融合分析」时走本文件，用户说「扫描」时走 genesis-scan。

---

## 四大分析框架

| 框架 | 权重 | 优先级 | 必须拉取的数据 |
|------|------|--------|--------------|
| **缠论周线** | **40%** | **最高(宪法级)** | `westock-data kline --period week` + `technical --group ma,macd` |
| 波浪理论 | 25% | 高 | `westock-data kline --period day --limit 500` |
| 板块轮动 | 20% | 中 | `westock-data sector --rank` + `profile` |
| 事件驱动 | 15% | 中 | `westock-data news --limit 15` |

---

## 宪法约束（最高优先级）

> **缠论看空 → 综合评分强制 ≤ 2.0，不做多。其他框架不能违背缠论方向。**

| 缠论评分 | 综合评分上限 | 操作建议 |
|---------|------------|---------|
| ★1（一卖） | ≤ 1.0 | 清仓 |
| ★2（二卖） | ≤ 2.0 | 严禁做多，轻仓做空 |
| ★3（震荡） | ≤ 3.0 | 轻仓观望 |
| ★4（一买） | ≤ 4.5 | 可做多，仓位 ≤ 50% |
| ★5（二买/三买） | 无上限 | 完美共振，重仓 |

**此约束无条件执行，不允许其他框架用高分覆盖。**

---

## 各框架评分标准（★1-5）

### 缠论周线

| 评分 | 信号 | 含义 |
|------|------|------|
| ★5 | 二买 / 三买 | 周线回调结束，趋势延续或新生 |
| ★4 | 一买 | 周线一买，次级别反弹可期 |
| ★3 | 上升中继 | 周线笔延续，震荡整理后新高 |
| ★2 | 一卖 / 二卖 | 周线顶背驰，反弹结束后出新笔 |
| ★1 | 三卖 | 周线下跌未结束 |

### 波浪理论

| 评分 | 信号 | 含义 |
|------|------|------|
| ★5 | 3浪主升 | 推动浪最强段，顺势持仓 |
| ★4 | 5浪起点 | 趋势尾声但未确认 |
| ★3 | 1浪 / 5浪中段 | 方向明确但非最强 |
| ★2 | 2浪 / 4浪调整 | 逆势，轻仓或观望 |
| ★1 | C浪 | 下跌未结束，清仓 |

### 板块轮动

| 评分 | 信号 | 含义 |
|------|------|------|
| ★5 | 龙头板块前5% | 领涨且有资金持续净流入 |
| ★4 | 强势板块前20% | 跟随龙头，动能强劲 |
| ★3 | 轮动板块 | 随大盘但有相对收益 |
| ★2 | 弱势板块 | 跑输大盘，轻仓 |
| ★1 | 资金持续流出 | 板块主跌，回避 |

### 事件驱动

| 评分 | 信号 | 含义 |
|------|------|------|
| ★5 | 重大政策催化 | 顶层政策 + 行业实质受益 |
| ★4 | 业绩超预期 | 财报大幅超越预期 |
| ★3 | 产品/订单催化 | 单一产品或订单驱动 |
| ★2 | 无明显事件 | 跟随板块 |
| ★1 | 重大利空 | 风险事件未完全定价 |

---

## 融合公式

```
综合评分 = 缠论分 × 0.40 + 波浪分 × 0.25 + 板块分 × 0.20 + 事件分 × 0.15
```

### 操作建议表

| 综合评分 | 操作 | 建议仓位 |
|---------|------|---------|
| 4.5 ~ 5.0 | 完美共振，重仓 | 80-100% |
| 4.0 ~ 4.4 | 强共振，可做多 | 60-80% |
| 3.0 ~ 3.9 | 弱共振，轻仓 | 30-50% |
| 2.0 ~ 2.9 | 观望为主 | 10-20% |
| < 2.0 | 清仓回避 | 0% |

## 数据拉取规范（每个框架必做）

### 缠论必须拉取
```bash
westock-data kline <symbol> --period week --limit 80
westock-data technical <symbol> --group ma,macd --date <latest>
```
**缺失处理**：数据不足 → 该框架评分强制 ≤ 2

### 波浪必须拉取
```bash
westock-data kline <symbol> --period day --limit 500
```
**缺失处理**：K线不足 → 该框架评分强制 ≤ 2

### 板块必须拉取
```bash
westock-data sector --search <行业关键词>
westock-data sector <板块代码>
westock-data asfund <symbol>
```
**缺失处理**：港股无板块数据 → 该框架权重均分给其他三个

### 事件必须拉取
```bash
westock-data news <symbol> --limit 15 --type 2
westock-data notice <symbol> --limit 5
```
**缺失处理**：无新闻数据 → 事件评分上限 ★3

## 分析执行流程

```
1. 宪法约束确认（M因子大盘方向，若M<4则整体降档）
2. 数据拉取（四个框架并行拉取，一次完成）
3. 逐框架评分（每个框架独立评分，不参考其他框架）
4. 融合计算（按权重公式加权）
5. 宪法约束检查（缠论≤2则强制综合分≤2.0）
6. 输出：
   - 四框架评分表
   - 综合评分 + 操作建议
   - 交易计划（止损/目标/仓位/触发条件）
   - 保存到 portfolio/plans/
```

## 多因子选股（候选池阶段）

在标的深入分析**之前**，先用 westock-tool 建立候选池：

```bash
# 价值+质量筛选
westock-tool filter "intersect([PE_TTM > 0, PE_TTM < 30, ROETTM > 15, Chg1Y > 20])" --limit 20 --orderby ROETTM:desc

# MACD金叉策略
westock-tool strategy macd_golden --limit 20

# 预置标签：新高突破
westock-tool strategy price_breakout_high --limit 20
```

筛选出的候选标的 → 进入 V22.1 四框架共振分析 → 融合评分排序 → 最终建议。

## 数据引用规范

分析报告中的数据必须标注来源：
- 周线MA值：westock-data kline `<symbol>` --period week
- MACD值：westock-data technical `<symbol>` --group macd
- 板块排名：westock-data sector --rank
- 新闻标题：westock-data news `<symbol>` --limit 15 --type 2
- 资金流向：westock-data asfund `<symbol>`

**禁止**：用 LLM 内部知识估算任何指标数值，必须真实拉取。
