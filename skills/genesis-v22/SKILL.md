# genesis-v22 — Genesis V24.1 Agentic 六框架裁决系统

> **版本**：V24.1  
> **版本对齐**：完全采用 V24.1 框架  
> **触发**：用户说「分析 XX」「综合评估」「融合分析」  
> **前置条件**：`westock-data` 必须可用

---

## 核心哲学转变

```
V22.1（本地旧版）：缠论独裁（缠论40%，其他不能覆盖）
V24.1（当前版）  ：Agentic 裁决（无单一否决者，M因子除外）
```

**为什么去掉缠论否决**：缠论是主观性框架，没有大规模回测验证，单独放在"不可违背"位置是系统性风险。波浪+缠论合并为"周期分析"保留参考价值，但不拥有否决权。

## 六大分析框架

| 框架 | 权重 | 类型 | 核心逻辑 | 取数命令 |
|------|------|------|----------|----------|
| 均值回归 | 18% | 技术 | BOLL下轨/RSI超卖/MACD底背离 → 超买超卖回归 | `technical --group boll,rsi,macd` |
| 海龟突破 | 18% | 技术 | Donchian通道突破 + ATR仓位 → 追趋势 | `kline --period day --limit 60` |
| 周期分析 | 14% | 技术 | 波浪理论 + 缠论笔/中枢 → 周期位置定位 | `kline --period week --limit 80` |
| CAN SLIM C/A | 16% | 基本 | EPS加速 + ROE + 营收CAGR → 盈利质量 | `finance --num 12` |
| CAN SLIM N/S/L/I | 16% | 基本 | 创新高 + 供需 + 龙头 + 机构认同 → 市场强度 | `kline` + `asfund` + `shareholder` |
| M因子（大盘）| **18%** | 宏观 | 指数vsMA50 + 市场广度 + 成交量 → 大盘方向 | `kline sh000300 --limit 60` |

**权重说明**：
- M因子看似18%，但它是**唯一拥有否决权的因子**——实际影响力远超18%
- 均值回归+海龟各18%，体现"回归vs趋势"这对经典对立范式的平衡
- 周期分析14%最低，因为浪型/缠论判定高度主观，两者互补后在内部对冲
- CAN SLIM 双组分占32%，基本面总权重最高

## 标准取数流程（分析任何标的前必须执行）

**⚠️ 数据源硬约束：不取数就编分析 = 违规**

```bash
# 必须命令（按顺序执行）
1. westock-data kline <代码> --period day --limit 260      # 日线
2. westock-data kline <代码> --period week --limit 80     # 周线
3. westock-data technical <代码> --group boll,rsi,macd    # 技术指标
4. westock-data finance <代码> --num 12                    # 财报
5. westock-data sector --rank interval_chg_rank_sw1 --sort chg5Days  # 板块排行
6. westock-data news <代码> --limit 15 --type 2          # 新闻

# 如需深入
7. westock-data asfund <代码>                              # 主力资金
8. westock-data shareholder <代码>                         # 十大股东
9. westock-data kline sh000300 --period day --limit 60    # M因子（沪深300）
10. westock-data chip <代码>                               # 筹码（仅沪深京A股）
```

**禁止行为**：
- ❌ 没跑 `kline` 就说"当前处于XX浪型"
- ❌ 没跑 `finance` 就说"ROE 为XX%"
- ❌ 没跑 `technical` 就说"RSI 超卖"
- ❌ 用缓存数据代替实时取数（除非标注"以下为X天前数据"）

## Agentic 裁决流程

```
Step 0: M因子快检（大盘否决权，唯一必做）
        M < 4 → 标注仓位上限，继续分析
        M < 2 → 建议直接回避，用户坚持可继续但必须标注

Step 1: 六框架并行数据拉取（一次完成，见上方6条命令）

Step 2: 六框架独立评估（每个框架不看其他框架结果）
        均值回归 → BOLL/RSI/MACD → 信号 + 方向 + 强度
        海龟突破 → 通道突破 + ATR → 信号 + 方向 + 强度
        周期分析 → 波浪浪型 + 缠论笔/中枢 → 信号 + 方向 + 强度
        CAN SLIM C/A → EPS/ROE/营收 → 评分
        CAN SLIM N/S/L/I → 新高/资金/机构 → 评分
        M因子 → 指数/广度/成交量 → 评分

Step 3: Agentic 裁决
        计算加权综合分
        多数一致性检查（≥60%看多/看空）
        分歧识别与披露
        M因子否决检查

Step 4: 仓位计算
        基础仓位 = f(综合分, 共识度)
        M因子上限 = f(M评分)
        最终仓位 = min(基础仓位, M因子上限)

Step 5: 输出裁决报告 + Self-Refutation
```

## 各框架评分标准（★1-5）

### 均值回归

| 评分 | 信号 | 条件 |
|------|------|------|
| ★5 | 强超卖 | RSI(6) < 20 + 价格在BOLL下轨下方 + MACD底背离 |
| ★4 | 超卖 | RSI(6) < 30 或 价格触及BOLL下轨 |
| ★3 | 中性 | RSI在40-60之间 |
| ★2 | 超买 | RSI(6) > 70 或 价格触及BOLL上轨 |
| ★1 | 强超买 | RSI(6) > 85 + 价格在BOLL上轨上方 + MACD顶背离 |

### 海龟突破

| 评分 | 信号 | 条件 |
|------|------|------|
| ★5 | 突破新高 | 价格突破20日新高 + 成交量放大1.5倍 |
| ★4 | 上升趋势 | MA20 > MA60 + 价格在两者上方 |
| ★3 | 横盘整理 | 无明显趋势 |
| ★2 | 下降趋势 | MA20 < MA60 + 价格在两者下方 |
| ★1 | 跌破新低 | 价格跌破20日新低 |

### 周期分析（波浪 + 缠论）

| 评分 | 信号 | 条件 |
|------|------|------|
| ★5 | 3浪主升 或 二买/三买 | 波浪3浪 + 缠论周线二买共振 |
| ★4 | 5浪起点 或 一买 | 波浪5浪起始 或 缠论周线一买 |
| ★3 | 1浪 / 5浪中段 | 趋势明确但非最强段 |
| ★2 | 2/4浪调整 或 二卖 | 逆势，轻仓 |
| ★1 | C浪 或 三卖 | 下跌未结束，清仓 |

### CAN SLIM C/A（盈利质量）

| 因子 | 阈值 | 得分 |
|------|------|------|
| EPS同比 | > 20% | 3分 |
| 营收同比 | > 15% | 2分 |
| EPS加速度 | 增速递增 | 2分 |
| 年度EPS增长 | > 25% | 3分 |
| ROE | > 15% | 3分 |
| 营收CAGR | > 15% | 2分 |

**C/A评分** = 得分之和/17 × 5 → 归一化到 ★1-5

### CAN SLIM N/S/L/I（市场强度）

| 因子 | 阈值 | 得分 |
|------|------|------|
| 52周接近度 | > 95% | 4分 |
| 突破放量 | > 1.5倍均量 | 3分 |
| 主力净流入 | > 0 | 3分 |
| 机构股东增加 | 是 | 3分 |

**N/S/L/I评分** = 得分之和/13 × 5 → 归一化到 ★1-5

### M因子（大盘方向）

| M评分 | 大盘状态 | 仓位上限 |
|-------|----------|----------|
| 8-10 | 强势 | 100% |
| 6-7 | 偏强 | 80% |
| 4-5 | 中性 | 60% |
| 2-3 | 偏弱 | **30%** ⚠️ |
| 0-1 | 弱势 | **10%** 🔴 |

## 多数否决制（宪法级）

| 多数方向 | 操作 |
|----------|------|
| ≥60%框架看多 | 可以建议买入 |
| 40-60%（无明显多数）| 轻仓/观望，必须披露分歧 |
| ≥60%框架看空 | **必须建议回避**，不管综合分多高 |

**分歧不是错误，是信息**。Agent 不掩盖分歧，而是作为最重要的风险提示披露。

---

## M因子否决权（宪法级）

**M因子是唯一拥有否决权的因子**。统计依据：欧奈尔核心发现——75%的个股走势跟随大盘方向。

M < 4 时，即使所有其他五个框架全部满分，最终仓位也不允许超过 30%。

## 融合评分公式

```
综合评分 = 均值回归×0.18 + 海龟×0.18 + 周期×0.14 + CAN_SLIM_CA×0.16 + CAN_SLIM_NSLL×0.16 + M因子×0.18
```

### 操作建议矩阵

| 多数方向 | 共识度 | 综合分 | 操作 | 基础仓位 |
|----------|--------|--------|------|----------|
| 看多 | ≥80% | ≥4.0 | 买入 | 60-80% |
| 看多 | 60-79% | 3.0-3.9 | 轻仓买入 | 30-50% |
| 看多 | <60% | 2.0-2.9 | 观察 | 10-20% |
| 无多数 | — | — | 观望 | 0-10% |
| 看空 | ≥60% | <2.0 | **回避** | 0% 🔴 |

**最终仓位 = min(基础仓位, M因子上限)**

## ATR止损与仓位计算

```
止损位 = 入场价 - ATR(14) × 2
风险金额 = 账户 × 2%
仓位 = 风险金额 / 止损幅度
```

---

## Self-Refutation（必须包含，至少3点）

1. **框架间分歧可能意味着**：哪个框架可能是错的？判断依据是否充分？
2. **时间窗口偏差**：结论在 ±20% 时间窗口内是否还成立？
3. **市场环境依赖**：策略在牛市/熊市/震荡市的失效边界是什么？
4. **数据局限**：哪些数据缺失可能影响判断？
5. **参数敏感性**：阈值（如RSI<30）是否经过鲁棒性验证？
