# genesis-orchestrator — Genesis V28 调度层

> **版本**：V28  
> **定位**：分析任务的智能调度中心。不是把所有东西塞一个文件，而是**按需调用**。  
> **核心原则**：不是每次都跑全部。不同任务用不同模式。  
> **触发**：所有「分析 XX」「扫描一下」「看看持仓」统一入口

---

## 三种分析模式

### 模式A：快速扫描（Mode-Fast）≈ 5分钟
> 用于：P0日常扫描 / 午间信号检查

```
Step 1: M因子快检（kline sh000300，1分钟）
Step 2: 板块热点（sector --rank，取前20，1分钟）
Step 3: 情绪温度（quote + 涨停数，1分钟）
Step 4: 新闻过滤（marketnews，取5条，1分钟）
Step 5: 黄金坑快速扫（跌幅>8%的标的，1分钟）
→ 输出：1页扫描报告，不超过200字
```

### 模式B：深度分析（Mode-Deep）≈ 20分钟
> 用于：首次关注某标的 / 持仓标的需要深度复盘

```
Step 1: 区间检测（genesis-regime，取当前相位）
Step 2: 标的类型识别（profile + quote）
Step 3: 五维数据采集（技术/基本面/资金/消息/宏观）
Step 4: 流派独立评估（加载对应流派skill，跑完）
Step 5: 量化严谨派审查
Step 6: 产业链验证
Step 7: 多情景压力测试
Step 8: 组合建议（如果涉及多只）
→ 输出：完整裁决报告
```

### 模式C：持仓追踪（Mode-Track）≈ 10分钟
> 用于：每日收盘后检查现有持仓

```
Step 1: 加载当前持仓（portfolio/positions.json）
Step 2: 拉取各持仓今日行情（quote）
Step 3: 检查止损/止盈是否触发
Step 4: 检查新事件（news）
Step 5: 区间相位有无变化
→ 输出：持仓追踪快报（<100字）
→ 若触发止损/止盈/区间变化 → 立即提醒用户
```

## 模式选择规则

```
用户说「扫描」「看看市场」「午间信号」→ Mode-Fast
用户说「分析XX」「帮我看看XX」「深度评估」→ Mode-Deep
用户说「看看持仓」「今天亏了吗」「我的股怎么样」→ Mode-Track
无明确指令但涉及持仓标的 → Mode-Track
新标的 + 无持仓 → Mode-Deep（首次）→ 后续→Mode-Track
```

## 流派 Skill 按需加载规则

> 每次分析只加载**相关**流派，不全加载。

```
标的类型 → 优先加载的流派：
────────────────────────────────────────
A.蓝筹大盘 → 基本面(fundamental) > 宏观(sector-rotation) > 趋势(trend-following)
B.成长题材 → 趋势(trend-following) > 动量(momentum-flow) > 事件(event-sentiment)
C.周期股   → 周期(elliott-wave) > 宏观(sector-rotation) > 基本面(fundamental)
D.价值低估 → 基本面(fundamental) > 回归(mean-reversion) > 资金(momentum-flow)
E.困境反转 → 事件(event-sentiment) > 情绪(news-signal) > 基本面(fundamental)
F.未知     → 全部流派快速扫一遍（但权重轻）
────────────────────────────────────────
所有标的 → 缠论(chan-theory) 默认加载（宪法级，权重最低20%）
所有标的 → 量化严谨派(quantitative-research) 必须加载（否决权）
```

## 板块池子机制

> 扫描后不是输出单个标的，而是分池子管理。

### 池子类型

| 池子 | 入池条件 | 用途 |
|------|---------|------|
| **🔥黄金坑池** | 跌幅8-15% + 横盘20日 + 止跌确认 | 等止跌后买入 |
| **📈早信号池** | 板块催化 + 涨幅3-8% + 多渠道确认 | 等回调5%后买 |
| **⚡材料短缺池** | 材料板块 + 供需催化 | 持有或加仓 |
| **🛡️防御池** | 低估值 + 高股息 + 抗跌 | 弱势市避险 |
| **🆕新关注池** | 深度分析完成，裁决≥3.5 | 等待买点 |

### 池子管理规则

```
入池：扫描发现 → 触发深度分析 → 裁决通过 → 进对应池子
出池：
  - 达到目标位 → 止盈出池
  - 止损触发 → 止损出池
  - 30天无进展 → 自动降级或出池
  - 裁决分连续2次 < 2.5 → 出池

每周池子审视：每周末跑 Mode-Fast，审视所有池子成员状态
```

---

## 板块分池扫描流程

```
1. 拉申万一级（按5日涨幅排序）
2. 识别当前强势板块（>5%）和弱势板块（<-3%）
3. 分流：
   - 强势板块 → 找等回调的标的 → 进📈早信号池
   - 弱势板块 → 找超跌的标的 → 进🔥黄金坑池
   - 有材料催化 → 进⚡材料短缺池
4. 输出：各池子更新报告
```

## 调度决策树

```
用户输入
   ↓
识别意图（扫描/分析/持仓/其他）
   ↓
选择模式（Fast/Deep/Track）
   ↓
识别标的类型（如有标的）
   ↓
按需加载流派skill
   ↓
执行分析
   ↓
更新池子（如涉及新标的）
   ↓
输出报告 + 归档到memory
```

## 输出格式规范

### Mode-Fast 输出（<200字）
```
【快扫】YYYY-MM-DD HH:MM

M因子:X/10 [仓位上限] | 强势板块:XX | 弱势板块:XX
🔥黄金坑: [标的, 跌幅, 信号]
📈早信号: [板块, 涨幅, 催化]
🆕关注: [标的, 原因]

[一句话建议]
```

### Mode-Deep 输出（完整报告）
```
【深度裁决】标的 (代码) | 类型:X | YYYY-MM-DD

## 一、区间背景（Phase X）
## 二、五维评分
## 三、流派解读
## 四、多情景
## 五、裁决
## 六、池子建议

[见 genesis-v22 完整模板]
```

### Mode-Track 输出（<100字）
```
【持仓追踪】YYYY-MM-DD HH:MM

持仓A: ±X% [正常/⚠️注意/🔴止损/🟢止盈]
持仓B: ±X% [状态]
...

⚠️ 需关注: [触发条件的标的]
```
